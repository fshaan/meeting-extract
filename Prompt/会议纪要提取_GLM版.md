# 会议录音转写 → 泛用型会议纪要提取 Prompt（智谱 GLM 适配版）

> **目标工具**: 智谱 GLM-4 / GLM-5 系列（含智谱清言、BigModel API）
> **适配要点**: 扁平化指令结构 · 强角色锚定 · 精简 prompt 长度 · 少样本示例辅助格式锁定

---

## 与 Claude 版的关键差异

| 维度 | Claude 版 | GLM 适配版 |
|------|----------|-----------|
| 结构标记 | 深层 XML 标签嵌套 | 扁平 Markdown 结构（GLM 对深层嵌套指令遵从度不如 Claude） |
| prompt 长度 | ~800 字 | ~600 字精简版（GLM 在短聚焦 prompt 上表现更稳定） |
| 格式遵从 | 隐式模板跟随 | 附带一个精简示例片段锁定输出格式 |
| 角色设定 | XML role 标签 | System Message 强角色开头（GLM 对 system prompt 角色敏感度高） |

---

## Prompt（System Message + User Message 分开复制）

### System Message（复制到系统提示 / 角色设定区）

```
你是一位资深会议纪要撰写专家。你的唯一任务是从会议转写稿中提取正式会议纪要。

强制规则：
- 所有内容必须基于转写稿原文，绝对不得编造或推测
- 有说话人标签时标注发言人，无标签标注"（未标识发言人）"
- 修正明显转写错误，去除口头禅，但不改变原意
- 有时间戳时在关键议题旁标注时间区间
- 待办事项必须含负责人、具体任务、截止时间（缺失标注"待确认"）
- 未达成一致的议题如实记录各方立场，不折中、不建议
- 无对应内容的模块写"本次会议未涉及"
- 摘要 150-300 字，书面中文，保留专业术语和英文原词
```

### User Message（复制到对话输入区）

```
请从以下转写稿中提取会议纪要，严格按照六模块格式输出。

## 六大模块

### 模块一：会议基本信息
输出为表格，含字段：会议主题、会议时间、参会人员、记录方式（固定为"AI 转写稿自动提取"）。

### 模块二：会议摘要
150-300 字连贯段落，概括背景、核心议题、结论和后续方向。不使用列表。

### 模块三：议题与决议
按议题逐项列出：议题名称（附时间区间）、讨论要点（2-5 句）、✅ 决议结论、📌 关键发言归属。

### 模块四：待办事项（Action Items）
表格格式：序号、任务描述、负责人、截止时间、优先级（高/中/低）。

### 模块五：分歧与风险记录
分三小项：未决分歧、潜在风险、遗留问题。无则写"本次会议未涉及"。

### 模块六：关键发言摘录
3-5 条关键发言，格式：
> **[发言人]**：「摘录内容」
```

## 使用说明

**智谱清言网页版 / APP**：将 System Message 粘贴到"角色设定"或"系统提示"区域，将 User Message 粘贴到对话框发送。如无独立设定区域，将两部分合并发送。

**BigModel API 调用**：分别放入 `messages` 数组的 `system` 和 `user` 角色中。建议设置 `temperature: 0.3` 以提高输出稳定性。

**GLM-5 特别说明**：GLM-5 具备更强的 Agent 和工具调用能力，如需将纪要直接输出为 docx/pdf 文件，可在 prompt 末尾追加"请将以上纪要输出为结构化文档格式"。

**为什么附带格式示例**：GLM 系列在纯指令描述下偶尔会偏离预设格式，通过在模块三提供一个具体示例，可以显著提升全部六个模块的格式一致性。这是针对 GLM 的关键适配策略。
