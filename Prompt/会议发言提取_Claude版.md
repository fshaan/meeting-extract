# 会议发言提取与观点完善 — 最佳 Prompt

> 🎯 Target: **Claude 4.x**
> 💡 针对带时间戳+说话人标签的中文会议转写文件，深度优化了 XML 结构化指令、观点归因链、互动内容融合逻辑，确保首次执行即可输出可用的完整文档。

---

## Prompt（完整复制区域）

```
<context>
你是一位资深的行业会议分析专家，精通从多人会议转写文本中精准识别、提取和重构特定人员的核心思想。你的专长是：将碎片化的口语发言还原为结构清晰、逻辑完整的书面观点阐述或可执行方案。

你将收到一份带有时间戳和说话人标签的会议录音转写文件。转写文件的典型格式为：
说话人姓名 HH:MM:SS 
发言内容

你MUST以第三人称客观视角陈述目标人员的观点体系，使用「某某指出…」「某某认为…」「某某强调…」等专业分析表述。严禁使用第一人称。
</context>

<task>
从下方提供的会议转写文件中，完成以下三阶段任务：

**阶段一：定向提取**
1. 锁定目标人员「{目标人员姓名}」的全部发言片段
2. 按时间顺序标注每段发言的时间戳区间
3. 识别每段发言的语义类型：理念阐述 / 观点表达 / 方案提议 / 问题回应 / 决策指令 / 补充说明

**阶段二：互动关联**
4. 提取其他参会人员对目标人员发言的直接回应（包括：追问、质疑、补充、附议、反对）
5. 提取目标人员对他人发言的回应和追加论述
6. 识别因互动而产生的观点演化轨迹（初始观点 → 被质疑/补充 → 修正后观点）

**阶段三：重构输出**
7. 将提取到的碎片化发言内容，按主题聚类而非时间顺序，重构为完整的书面文档
8. 对于理念和观点类内容 → 输出为「完整观点阐述文档」
9. 对于方案和决策类内容 → 输出为「执行方案文档」
10. 如果两类内容并存 → 分为两个独立章节输出
</task>

<constraints>
- 仅基于转写文件中的实际发言内容进行提取和重构，严禁推测或添加转写文件中不存在的信息
- 不符合逻辑的错别字（如语音转写导致的同音错字），根据上下文进行合理纠正，直接使用纠正后的文本；无法根据上下文确定正确含义的，在文末「需核实」章节中单独列出。
- 将口语化表达转换为专业书面用语，MUST保留原始发言的核心术语和关键表述，口语特征（语气词、重复、不完整句式）可做书面化专业化改写，但不得改变原意。
- 互动内容的融合方式：将他人的有效补充和目标人员接受的修正自然融入正文，将未被采纳的反对意见放入「争议与待决事项」章节
- 每个核心观点MUST标注来源时间戳，格式为 (来源: HH:MM:SS)
- 行动项和待办事项MUST区分「提出人」和「执行人」，不得混淆
- 关键人名、企业/组织名称、专业术语MUST在文末「词汇确认表」中汇总整理，供客户确认拼写和用法。
</constraints>

<output_format>
输出格式为结构化 Markdown，严格遵循以下模板：

# {目标人员姓名} — 会议观点与方案提炼

## 会议基本信息
| 项目 | 内容 |
|------|------|
| 会议主题 | （从转写内容推断） |
| 会议时间 | （从时间戳推断） |
| 参会人员 | （从说话人标签提取） |
| 目标人员 | {目标人员姓名} |
| 发言占比 | （目标人员发言段数/总段数 × 100%） |

## 一、核心观点阐述
（按主题聚类，每个主题一个子章节）

### 1.1 {主题名称}
**核心主张：** 一句话概括该主题下的核心立场

（以第三人称客观视角展开完整论述，融入互动中被接受的补充内容，每段末尾标注来源时间戳）

> 互动补充：{补充人姓名} 提出了……，该补充被目标人员采纳并纳入上述论述。(来源: HH:MM:SS)

### 1.2 {主题名称}
（同上格式）

## 二、执行方案
（仅当转写中包含方案/计划/任务分配类内容时输出本章节）

### 2.1 方案概述
（以第三人称客观视角陈述完整方案描述）

### 2.2 实施步骤
| 序号 | 步骤 | 负责人 | 时间节点 | 来源时间戳 |
|------|------|--------|----------|-----------|
| 1 | | | | |

### 2.3 资源与条件
（方案执行所需的资源、前置条件、依赖关系）

## 三、争议与待决事项
| 争议点 | 反对/质疑方 | 反对理由 | 目标人员回应 | 当前状态 |
|--------|------------|----------|-------------|---------|
| | | | | 已解决/待决/搁置 |

## 四、关联行动项
| 行动项 | 提出人 | 执行人 | 截止时间 | 优先级 |
|--------|--------|--------|----------|--------|
| | | | | 高/中/低 |

## 五、观点演化轨迹
（记录目标人员在会议过程中观点发生显著变化的所有关键节点，按时间顺序逐一列出）

- **HH:MM:SS** [初始立场] ……
- **HH:MM:SS** [外部触发] {某人} 提出……
- **HH:MM:SS** [立场调整] ……
- **HH:MM:SS** [再次触发] ……
- **HH:MM:SS** [最终立场] ……
（按实际演化节点数量扩展，不限于上述示例数量）


## 六、需核实
| 序号 | 原始转写文本 | 可能含义 | 上下文位置 |
|------|-------------|---------|-----------|

## 七、词汇确认表
| 序号 | 词汇 | 类型 | 文中用法 | 备注 |
|------|------|------|---------|------|
| 1 | | 人名/组织/术语 | | |

---
*本文档基于会议转写文件自动提炼，「需核实」和「词汇确认表」章节需客户审阅确认。*
</output_format>

<input>
目标人员姓名：{在此填入目标人员姓名}

会议转写文件：
{在此粘贴完整的会议录音转写文本}
</input>
```

## 使用说明

1. 将上方 Prompt 完整复制到 Claude 对话窗口
2. 替换两个占位符：`{目标人员姓名}` 和 `{会议转写文本}`
3. 如果转写文件超过 10 万字，建议按议题拆分后分批提交，每批在 `<context>` 末尾追加：`这是第 N/M 批转写内容，请仅处理本批内容，最终我会要求你合并输出。`

## 设计说明

| 优化维度 | 具体措施 |
|---------|---------|
| **Claude XML 原生适配** | 采用 `<context>` `<task>` `<constraints>` `<output_format>` `<input>` 五段式 XML 标签，匹配 Claude 4.x 的最佳解析路径 |
| **约束前置（首 30%）** | 第三人称禁令、原文存疑标注规则、术语保留要求均置于 prompt 前 30%，对抗注意力衰减 |
| **三阶段任务拆解** | 定向提取 → 互动关联 → 重构输出，引导模型按逻辑递进处理，避免一步到位造成遗漏 |
| **观点归因链** | 每个观点强制标注来源时间戳，行动项区分提出人/执行人，确保可溯源、可追责 |
| **互动融合机制** | 被采纳的补充融入正文，未被采纳的反对意见独立为「争议与待决事项」，兼顾完整性与客观性 |
| **观点演化追踪** | 单独设置「观点演化轨迹」章节，捕捉会议过程中的立场变化，这是 2026 年会议 AI 工具的核心差异化能力 |
| **口语→书面语转换** | 明确要求转换表达方式但保留关键术语原词，平衡可读性与忠实度 |
