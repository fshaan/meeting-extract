# v0.3.0 Prompt 模板优化设计文档

> **版本**: v0.3.0
> **日期**: 2026-04-08
> **状态**: 待实施
> **影响范围**: 8 个 prompt 模板文件（会议纪要 ×4 + 发言提取 ×4）

---

## 背景

v0.2.0 对 Gemini/GLM/Qwen 三版模板做了多项改进（第三人称视角、词汇确认表、需核实章节、去"模块"标签），但同时引入了两个质量退化问题：

1. **会议纪要**：书面化规则过于激进（"完全转化"），导致 AI 在议题拆分上趋于合并、摘要概括化；时间锚定规则被直接删除而非内部化。
2. **发言提取**：观点演化轨迹从多节点带时间戳格式简化为仅 3 行模板，导致输出从详细演化记录退化为笼统叙述；来源时间戳被全面移除。

Claude 版模板仍停留在 v0.1.0，与其他三版存在结构性分歧。

## 目标

- 纠正 v0.2.0 的质量退化（项 1-5）
- 将 Claude 版升级为 v0.3.0 基准（项 7-9）
- 缓和过度严格的约束措辞（项 6）
- 确保 4 个模型版本的结构和规则一致性

## 约束

- 修改范围仅限 prompt 模板内容，不涉及脚本或工作流
- Claude 版先升级为基准，其他三版基于它级联适配
- 版本号升至 v0.3.0

---

## 执行策略

**方案 A：Claude-first 级联**

1. 先完成 Claude 版全部改动（项 1/2/3/5/7/8/9）
2. 基于 Claude 版级联更新 Gemini/GLM/Qwen（项 1/2/3/4/5/6）

选择理由：Claude 版是单一事实来源，先稳定基准再级联适配，避免返工。

## 9 项修改明细

### 项 1：回退"完全转化"为柔性书面化要求

**优先级**: 高 | **影响**: 会议纪要 ×4

#### Claude 版

`<instructions>` 核心规则第 3 条：

```diff
- 去除口头禅、重复语句和填充词，但不得改变发言人的原始立场和表达意图。
+ 去除口头禅、重复语句和填充词，将口语化表达整理为书面用语，但保留发言人的原始用词习惯和关键表述，不得改变原始立场和表达意图。
```

#### Gemini 版

核心规则第 3 条：

```diff
- 去除口头禅、重复语句和填充词，将口语化表达完全转化为专业书面用语，但不得改变发言人的原始立场和表达意图。
+ 去除口头禅、重复语句和填充词，将口语化表达整理为书面用语，但保留发言人的原始用词习惯和关键表述，不得改变原始立场和表达意图。
```

质量约束：

```diff
- 使用书面中文，将全部口语化表达转化为专业书面用语
+ 使用书面中文，保留发言人原始用词和专业术语
```

#### GLM 版

system 强制规则第 3 条：同 Gemini 版改法。

system 末尾质量行：

```diff
- 将全部口语化表达转化为专业书面用语
+ 保留发言人原始用词和专业术语
```

#### Qwen 版

system 强制规则第 3 条：同 Gemini 版改法。

system 输出质量：

```diff
- 将全部口语化表达转化为专业书面用语，保留原文专业术语和英文专有名词
+ 使用书面中文，保留发言人原始用词、专业术语和英文专有名词
```

### 项 2：时间锚定改为内部处理指令

**优先级**: 高 | **影响**: 会议纪要 ×4

#### Claude 版

`<instructions>` 核心规则第 4 条：

```diff
- 当转写稿包含时间戳时，在关键议题和重要决议旁标注对应时间区间，方便回溯原始录音。
+ 当转写稿包含时间戳时，利用时间信息辅助定位发言归属和议题边界，但输出中不显示时间戳。
```

`<output_format>` 议题与决议：

```diff
- - **议题名称**（附时间区间，如有）
+ - **议题名称**
```

#### Gemini/GLM/Qwen 版

v0.2.0 已删除时间锚定规则。在各版核心规则中新增一条（插入在书面化规则之后）：

> 当转写稿包含时间戳时，利用时间信息辅助定位发言归属和议题边界，但输出中不显示时间戳。

---

### 项 3：观点演化轨迹恢复多节点带时间戳格式

**优先级**: 高 | **影响**: 发言提取 ×4

#### 全部 4 版统一改为

```markdown
## 五、观点演化轨迹
（记录目标人员在会议过程中观点发生显著变化的所有关键节点，按时间顺序逐一列出）

- **HH:MM:SS** [初始立场] ……
- **HH:MM:SS** [外部触发] {某人} 提出……
- **HH:MM:SS** [立场调整] ……
- **HH:MM:SS** [再次触发] ……
- **HH:MM:SS** [最终立场] ……
（按实际演化节点数量扩展，不限于上述示例数量）
```

Claude 版原有 3 节点格式扩展为上述多节点格式。
Gemini/GLM/Qwen 版从无时间戳 3 行格式恢复为上述格式。

---

### 项 4：恢复来源时间戳到核心观点/互动补充/实施步骤

**优先级**: 高 | **影响**: 发言提取 ×4

#### Claude 版

已有 `<constraints>` 第 5 条时间戳要求，无需改动。

#### Gemini/GLM/Qwen 版

**强制规则新增**：

> 每个核心观点MUST标注来源时间戳，格式为 (来源: HH:MM:SS)

**模板修改**（三版相同）：

核心观点段落说明：

```diff
- （以第三人称客观视角陈述完整论述，融入互动中被接受的补充内容）
+ （以第三人称客观视角陈述完整论述，融入互动中被接受的补充内容，每段末尾标注来源时间戳）
```

互动补充格式：

```diff
- > 互动补充：{补充人姓名} 提出了……，该补充被目标人员采纳并纳入上述论述。
+ > 互动补充：{补充人姓名} 提出了……，该补充被目标人员采纳并纳入上述论述。(来源: HH:MM:SS)
```

实施步骤表头：

```diff
- | 序号 | 步骤 | 负责人 | 时间节点 |
+ | 序号 | 步骤 | 负责人 | 时间节点 | 来源时间戳 |
```

### 项 5：用词规则改为"保留核心术语，口语特征可专业化改写"

**优先级**: 高 | **影响**: 发言提取 ×4

#### Claude 版

`<constraints>` 第 3 条：

```diff
- 将口语化表达转换为书面语，但MUST保留原始发言的核心用词和关键术语，不得替换为同义词
+ 将口语化表达转换为专业书面用语，MUST保留原始发言的核心术语和关键表述，口语特征（语气词、重复、不完整句式）可做书面化专业化改写，但不得改变原意。
```

#### Gemini/GLM/Qwen 版

各版 Hard Constraints / 强制规则中对应条目：

```diff
- 将全部口语化表达完全转化为专业的书面用语，不必保留原始用词和语气特征。
+ 将口语化表达转换为专业书面用语，MUST保留原始发言的核心术语和关键表述，口语特征（语气词、重复、不完整句式）可做书面化专业化改写，但不得改变原意。
```

### 项 6：模板约束缓和（去掉"任何"极端词）

| Qwen | 强制规则第7条 | 「**任何**模板之外的前言、后记或解释」 | 「模板之外的前言、后记或解释」 |

### 项 7：Claude 版发言提取改第三人称

**优先级**: 中 | **影响**: 发言提取 Claude 版

`<context>` 末尾：

```diff
- 你MUST以第一人称视角还原目标人员的观点体系，严禁使用"某某提出…""某某认为…"等第三人称转述话术。
+ 你MUST以第三人称客观视角陈述目标人员的观点体系，使用「某某指出…」「某某认为…」「某某强调…」等专业分析表述。严禁使用第一人称。
```

`<output_format>` 中所有视角描述：

```diff
- （以第一人称视角展开完整论述……）
+ （以第三人称客观视角展开完整论述……）

- （以第一人称还原完整方案描述）
+ （以第三人称客观视角陈述完整方案描述）
```

### 项 8：Claude 版会议纪要去"模块"标签 + 加词汇确认表

**优先级**: 中 | **影响**: 会议纪要 Claude 版

`<task>` 中：

```diff
- 严格按照 <output_format> 中定义的六大模块输出。不得遗漏任何模块，不得自行新增模块。
+ 严格按照 <output_format> 中定义的各章节输出。不得遗漏任何章节，不得自行新增章节。
```

`<output_format>` 章节标题：

```diff
- ## 模块一：会议基本信息
+ ## 会议基本信息

- ## 模块二：会议摘要
+ ## 会议摘要

- ## 模块三：议题与决议
+ ## 议题与决议

- ## 模块四：待办事项（Action Items）
+ ## 待办事项（Action Items）

- ## 模块五：分歧与风险记录
+ ## 分歧与风险记录

- ## 模块六：关键发言摘录
+ ## 关键发言摘录
```

在关键发言摘录之后新增：

```markdown
## 词汇确认表

（汇总文中出现的关键人名、企业/组织名称、专业术语，供客户确认拼写和用法）

| 序号 | 词汇 | 类型 | 文中用法 | 备注 |
|------|------|------|---------|------|
| 1 | | 人名/组织/术语 | | |
```

`<instructions>` 核心规则新增第 7 条：

> 关键人名、企业/组织名称、专业术语在文末「词汇确认表」中汇总整理，供客户确认拼写和用法。

`<instructions>` 质量约束（与项 1 联动）：

```diff
- 使用书面中文，避免口语化表达
+ 使用书面中文，保留发言人原始用词和专业术语
```

### 项 9：统一错别字处理策略为"智能纠正 + 需核实"

**优先级**: 中 | **影响**: 发言提取 Claude 版

`<constraints>` 第 2 条：

```diff
- 当转写文件中存在语音识别错误或语义不清的片段时，用 [原文存疑：原始文本] 标注，不要擅自修正语义
+ 不符合逻辑的错别字（如语音转写导致的同音错字），根据上下文进行合理纠正，直接使用纠正后的文本；无法根据上下文确定正确含义的，在文末「需核实」章节中单独列出。
```

`<output_format>` 在观点演化轨迹之后新增：

```markdown
## 六、需核实
| 序号 | 原始转写文本 | 可能含义 | 上下文位置 |
|------|-------------|---------|-----------|
| 1 | | | |

> 若全文无需核实项，写"本次提取无需核实内容"
```

同步新增词汇确认表章节：

```markdown
## 七、词汇确认表
| 序号 | 词汇 | 类型 | 文中用法 | 备注 |
|------|------|------|---------|------|
| 1 | | 人名/组织/术语 | | |

---
*本文档基于会议转写文件自动提炼，「需核实」和「词汇确认表」章节需客户审阅确认。*
```

`<constraints>` 新增词汇确认规则：

> 关键人名、企业/组织名称、专业术语MUST在文末「词汇确认表」中汇总整理，供客户确认拼写和用法。
