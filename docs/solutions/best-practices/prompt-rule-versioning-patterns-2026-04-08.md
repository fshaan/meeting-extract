# Prompt 规则版本迭代的两个反模式及修正策略

## Context

v0.2.0 对 8 个会议转写 prompt 模板做了多项修改，实际输出对比发现两类质量退化：
1. AI 在议题拆分上趋于合并（信息颗粒度降低），摘要变得概括而非详实
2. 观点演化轨迹从 9 个带时间戳的详细节点退化为一段笼统叙述

根因分析发现退化并非功能设计问题，而是 prompt 规则的措辞策略导致 AI 行为偏移。

## Guidance

### 反模式一：删除规则而非内部化

v0.2.0 为了让输出更简洁，直接删除了「时间锚定」规则（要求在关键议题旁标注时间区间）。结果 AI 不仅不输出时间戳，还失去了利用时间信息定位发言归属的能力，导致议题边界识别变差。

**修正策略：内部化而非删除**

```diff
- （删除整条时间锚定规则）
+ 当转写稿包含时间戳时，利用时间信息辅助定位发言归属和议题边界，但输出中不显示时间戳。
```

规则仍然存在，AI 仍然使用时间信息进行内部推理，只是不在输出中显示。

### 反模式二：用绝对化措辞替代柔性引导

v0.2.0 将「去除口头禅，整理为书面用语」改为「将全部口语化表达完全转化为专业书面用语，不必保留原始用词」。「完全」「全部」「不必保留」的叠加导致 AI 过度改写：连专业术语和关键表述也被替换为同义词，丢失了原始发言的信息密度。

**修正策略：柔性结构 = 保留什么 + 可改写什么**

```diff
- 将全部口语化表达完全转化为专业书面用语，不必保留原始用词和语气特征。
+ 将口语化表达转换为专业书面用语，MUST保留原始发言的核心术语和关键表述，
+ 口语特征（语气词、重复、不完整句式）可做书面化专业化改写，但不得改变原意。
```

明确告诉 AI「保留什么」和「可以改写什么」，比用绝对化词语约束「做什么」更稳定。

## Why This Matters

Prompt 模板的措辞微调看似无害，但对 AI 输出质量的影响是非线性的。一个「完全」或一条规则的删除就能导致输出从「详实、结构化」退化为「概括、笼统」。当模板需要跨 4 个模型同步维护时，这种退化会被放大。

## When to Apply

- 需要简化 prompt 输出格式时（不要删规则，改为内部化）
- 需要调整 AI 的改写力度时（不要用绝对词，用柔性结构）
- 跨多模型同步修改 prompt 时（先改基准模板再级联）

## Examples

**模板输出示例对比：**

| 维度 | v0.2.0（绝对化规则） | v0.3.0（柔性规则） |
|------|---------------------|-------------------|
| 议题拆分 | 3 个合并议题 | 7 个独立议题 |
| 专业术语 | 被替换为通用同义词 | 保留原文用词 |
| 观点演化 | 3 行笼统叙述 | 5-9 个带时间戳节点 |
| 时间定位 | 无（规则已删除） | 内部使用，辅助归因 |

**跨模型同步策略：Claude-first 级联**

1. 先将 Claude 版升级为基准（它有 XML 标签的天然结构化优势）
2. 基于 Claude 版级联适配 Gemini/GLM/Qwen（各自保留模型特定的适配策略）
3. 用 grep 交叉验证确保所有版本规则一致
